# Changelog (2026/05/12)

## Added

- Added a template switch to `sensors/binary/ttp223.yaml`.
- The TTP223 binary sensor now toggles the new switch on each touch, allowing
  Home Assistant automations to use the module like a simple touch toggle.
- Added `This package exposes` header blocks across board, common, peripheral,
  and sensor package files.
- Added or normalized ESPHome component reference links in package headers.
- Added a modernization audit for the current package set.
- Added a git-ignored `local_workbench/` for private device and package
  experiments.
- Documented the approved header section order using `Bill of Materials` for
  physical build hardware lists.
- Added first-pass sensor packages for CT clamp, MiCS-5524, SparkFun weather
  station sensors, YF-B10 flow, BLE tracker, AS3935, INA219, keypad, VL53L0X,
  I2S audio, AJ-SR04M, and GPS.
- Added starter templates under `sample_projects/board_templates/`.

## Changed

- Breaking change: changed the ESP32 S3 Super Mini board UART bus0 defaults from
  GPIO3 TX / GPIO4 RX to GPIO35 TX / GPIO48 RX.
- Documented that public sample projects should default to
  `common/network/wifi_dynamicip.yaml`, with fixed IP kept as an intentional
  opt-in.
- Corrected the TTP223 user-facing name from `Capasitive` to `Capacitive`.
- Updated the TTP223 module references to point at the GPIO binary sensor and
  template switch ESPHome components.
- Added the missing header to `common/time/gps.yaml`.
- Normalized active package headers from `CCO` to `CC0`.
- Corrected stale usage examples that pointed at old `common/...`,
  `board/...`, or `sensors/digital/...` paths.
- Corrected BME280 labels that still said `BMP280`.
- Corrected SGP30 naming that still said `SPG30`.
- Moved new starter templates out of the root `template/` folder and into
  `sample_projects/board_templates/`.
- Normalized new sensor filenames to lowercase underscore naming.

## Fixed

- Fixed `$...` substitution placeholders in:
  - `sensors/analogue/tds_sensor.yaml`
  - `sensors/i2c/ltr390.yaml`
  - `sensors/i2c/sgp40.yaml`
- Cleared the inherited voltage device class from the TDS ppm output sensor.

## Validation

- Generic YAML parsing passed for the changed files.
- ESPHome `config` validation passed with a temporary scratch ESP32
  configuration that included `sensors/binary/ttp223.yaml`.
- ESPHome `config` validation passed with a temporary scratch ESP32
  configuration that included the updated TDS, LTR390, and SGP40 packages.
- ESPHome `config` and `compile` validation passed for the private minimal
  validation sample using the ESP32 S3 Super Mini board package.
- Physical boot test passed for the ESP32 S3 Super Mini minimal validation
  sample; WiFi and the captive portal were visible. Network join remains
  untested.
- Generic YAML parsing passed for the current sensor package set, sample
  templates, and private workbench device files.
