# Changelog (2026/05/28)

## Changed

- Cleaned the public documentation boundary by removing development-only
  scaffolding from the published tree.
- Removed deferred component documentation from the public documentation set.
  The preserved hardware information component remains local-only under
  `internal_documentation/` for later review.
- Updated the header standard to include compatibility status and validated
  ESPHome version fields.

## Added

- Added peripheral compatibility documents split by package type:
  - displays
  - speakers
- Added sensor compatibility documents split by package type:
  - analogue
  - binary
  - Bluetooth
  - I2C
  - I2S
  - One Wire
  - UART
- Added a shared compatibility status standard:
  - Not tested
  - Needs validation
  - Experimental
  - Tested
- Added the compatibility status legend to each compatibility document so the
  meaning of each status is visible on the page being read.
- Added the first draft of the beginner getting started guide for the Sensor to
  Dashboard workflow.

## Notes

- `Tested` now requires the ESPHome version used during validation.
- `Needs validation` explicitly means AI-assisted review and ESPHome
  compilation/config validation found no blocking errors, but a person has not
  validated the package on physical hardware yet.
- Initial compatibility entries are conservative. Packages are marked as
  `Not tested` unless this rebuild has compilation evidence.
- SendSpin PCM5102 remains marked `Experimental` because it currently depends on
  ESPHome development functionality during validation.
- Follow-up cleanup noted during documentation work:
  - `sensors/uart/gps.yaml` appears to have a mismatched header title.
  - `sensors/i2c/matrix_keypad_3x4.yaml` may need a better long-term folder
    home because it is a GPIO matrix keypad package rather than a typical I2C
    sensor.
