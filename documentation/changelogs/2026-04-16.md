# Latest Updates (2026/04/16)

Tested with ESPHome 2026.4.

Google Jules was run against the original repository. It found a few issues that
were fixed, but its ESPHome knowledge appeared to be several months out of date.
Treat AI-generated ESPHome changes with caution and verify against current
ESPHome documentation and real builds.

In short, the WiFi and a few other modules are being rewritten to cater for
deployments outside the home. This means an ESP32 package should be able to
connect to a WiFi access point using Improv or the captive portal.

Another update is coming soon.

   - Expanded safe mode.
   - Added a safe mode button.

   - Added various other improvements and cleanup.
